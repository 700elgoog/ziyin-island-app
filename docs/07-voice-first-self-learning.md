# 字音岛 v0.4：全程语音自学重构

> 本文记录 v0.4 方案。v0.5 已按真机反馈删除底部语音栏，并改为核心元素直接点读；现行规则以 `08-core-element-voice-map.md` 为准。

## 重构目标

目标用户是中国大陆 5—6 岁、尚不能稳定阅读界面文字的儿童。完成标准不是“页面上写了说明”，而是儿童只听语音、看动作提示，也能知道当前要做什么、如何操作、做对或做错以后怎么办。

## 从参考产品提炼的方法

本项目只借鉴教学与交互方法，不复制参考产品的角色、美术、音频、文案或具体游戏。

| 方法 | 字音岛实现 |
|---|---|
| 固定学习节奏 | 每节课固定为“玩—认—调—说—练—写”六段，儿童可预期下一步 |
| 进入即讲 | 每个独立页面打开后自动播放一句任务说明 |
| 一屏一事 | 六段课程每屏只突出一个主要动作 |
| 先示范后操作 | 先看原创口型、听本地标准音，再录音或选择 |
| 随时重听 | 全局底栏始终保留大喇叭；播放时同一位置变为暂停 |
| 即时反馈 | 选择、节奏、描写、跟读、识字、故事翻页都有成功或重试语音 |
| 儿童不依赖文字 | 文字继续保留给家长和无障碍使用，但图形、位置、颜色和语音可以独立完成任务 |

## 六环节课程

1. 玩：点击会变化的声音目标，先建立注意和兴趣。
2. 认：显示大字形、原创口型卡，并播报嘴唇、舌位、气流口令。
3. 调：单韵母练四声；其他课程练声音组合或整体认读。
4. 说：先播放本地标准音，再录下儿童跟读并回听。
5. 练：听音后从大卡片中选择，答错时不给压力，重新提示关键差异。
6. 写：进入大面积触控描写，完成后给予语音鼓励并保存课程进度。

## 发音数据覆盖

`PronunciationGuidance` 为 6 个单韵母、23 个声母、18 个复/鼻韵母、整体认读和拼读目标提供儿童化口令。每条至少包含：

- 嘴唇/张口提示；
- 舌位提示；
- 气流或送气提示；
- 可以直接播报给儿童的完整口令；
- 对应的口型视觉类别。

例如 `a` 的引导为：看嘴型，嘴巴张大；舌头放平、位置低；气流顺顺出来；再跟读“啊——，a”。这类口令是教学脚本，正式发布前应由普通话教师和儿童语言教师审校。

## 声音分层和互斥

| 声音 | 当前来源 | 使用规则 |
|---|---|---|
| 页面任务引导 | Android 中文 TTS | 只讲“现在做什么”，进入自动播报，可暂停和重听 |
| 拼音标准读音 | 本地 MP3 | 只承担音素/音节示范，绝不由 TTS 替代 |
| 儿童录音 | 本机麦克风与本地文件 | 录音时停止引导和标准音；播放回听时也停止引导 |
| 游戏反馈/故事 | 中文 TTS | 操作后播报；新提示会替换旧提示，避免重叠 |

引擎使用短时音频焦点。进入录音、播放标准音或回听录音时，全局引导立即停止。页面切换后，旧页面语音不会继续播放。

## 页面覆盖清单

欢迎页、首页、声音森林、拼音地图、六段拼音课、声调游戏、描写、汉字小镇、汉字学习、复习游戏、故事剧场、故事逐页阅读、成长花园和家长中心均有进入语音。儿歌选择和节拍、课程阶段选择、声调答案、录音结果、描写开始/完成、识字分类/词语、复习答案、故事每一页以及家长门也有操作语音。

## Image2 资产

新增 `mouth_a/o/e/i/u/v_v1.png` 六张原创 1024×1024 口型卡，放在 Android `drawable-nodpi`。它们只帮助儿童观察嘴唇和张口程度；舌位与气流由语音脚本补充，不能把 AI 图当作权威发音图。正式发布前需逐张教师复核。

## 自动检查

- 每个课程符号必须返回非空的嘴型、舌位、气流和儿童口令。
- 六个核心元音必须使用六种独立口型视觉。
- 除六段课程内部引导外，每个独立页面必须有进入语音。
- 六张 Image2 口型卡必须存在、为方形且不小于 1024 像素。

## 商用前仍需完成

当前页面引导使用设备中文 TTS，适合功能验证。正式儿童产品应邀请配音员录制全部引导并以本地音频包替换 TTS，同时保留现有脚本键值作为音频索引。拼音教学脚本、口型卡和标准音必须经过普通话教师终审和目标儿童可用性测试。
